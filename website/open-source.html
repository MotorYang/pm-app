开源透明

Pomo 是一个完全开源的项目。我们相信透明和社区协作是构建值得信赖的软件的基础。 每一行代码都可以被审计，每一个功能都欢迎社区贡献。

查看 GitHub 仓库

为什么开源？

对于处理敏感数据的工具，开源不仅是一种开发模式，更是一种责任。

完全透明

每一行代码都是公开的。你可以准确地知道应用在做什么， 数据如何处理，加密如何实现。没有隐藏的后门，没有秘密的数据收集。

安全可审计

安全不应该依赖于"模糊"。开源代码可以被安全研究人员、 社区成员和任何感兴趣的人审计，及时发现并修复潜在漏洞。

社区驱动

开源项目属于社区。用户可以报告问题、提出建议、贡献代码。 Pomo 的发展方向由真正使用它的开发者决定。

技术栈

参与贡献

我们欢迎各种形式的贡献：代码、文档、问题报告、功能建议，或者只是分享给其他开发者。

1

Fork 仓库

在 GitHub 上 Fork Pomo 仓库到你的账号

2

创建分支

基于 develop 分支创建你的功能分支

3

提交更改

编写代码并提交，确保遵循项目的代码规范

4

发起 PR

创建 Pull Request，描述你的更改内容

开始贡献

其他参与方式

不写代码也可以为项目做出贡献

报告问题

发现 Bug？在 GitHub Issues 中报告，帮助我们改进产品。详细的复现步骤会非常有帮助。

功能建议

有好的想法？在 GitHub Discussions 或 Issues 中分享你的功能建议和使用场景。

完善文档

帮助改进文档、翻译内容，或者写教程文章介绍如何使用 Pomo。

开源许可

Pomo 采用 MIT 许可证发布。你可以自由使用、修改和分发， 包括用于商业目的。唯一的要求是保留版权声明。
